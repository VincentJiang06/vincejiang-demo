# REACTOR 知识树排版算法 v4（正式规格）

> 2026-07-19 立项。背景：v1–v3 的布局是手排 tier + 网格摆放的启发式，行内穿插、
> 间隙失控、支线与主线不分、案例/汇流悬远等问题反复出现——根源是没有真正的
> 布局算法。本规格定义一个基于 Sugiyama 分层框架的确定性排版器，从 prereq 图
> **自动推导**全部几何。实现 = `site/lib/tree-layout.mjs`，由 build.mjs 调用；
> tree.json 的 `tier` 字段从此**废弃**（层由图自身决定），仅保留少量语义钉
> （`slot` 用于汇流舱叙事序等）。

## 0. 输入 / 输出契约

输入：`nodes[{id, branch, prereqs[]}]` + 常数（芯片宽高、最小间隙、区序）。
输出：每节点 `{x, y}` + `levels[{y,label}]` 分层线 + 校验报告。
**硬不变量（违反即 build 失败）**：
- I1 同层绝不穿插错位（一层一行，行内单一 y）
- I2 任意两芯片水平间隙 ≥ GAP_X(48px)、垂直走廊 ≥ GAP_Y(110px)
- I3 理论节点不出自己分支的水平区间（簇约束）
- I4 理论区内所有边严格向下；带区允许少量同层横边
- I5 输出确定性（同输入必同输出，无随机）

## 1. 分区（宏观结构，用户裁决固化）

自上而下：ROOT → 理论区（红|蓝|黄三纵簇）→ 案例带（横贯）→ 防御带（横贯）
→ 汇流舱（横条卡一行）。绿支不再是并列纵列；C 系不再是末端挂坠。
每区之间画横穿虚线 + LEVEL 标签（入门/进阶/深水区/案例带/防御与设计/汇流）。

## 2. 五阶段流水线

**S1 分层（layer assignment）**
- 理论区：最长路径分层 `down(n) = 1 + max(down(p))`；随后**松弛下沉**：
  对「消费者都在很深层」的节点（入门桥类），把层拉到
  `min(layer(d)) - 1`（d=全部依赖者），使桥卡恰好悬在首个消费者上方——
  Y16/Y17/B15/B17/R15 的位置由此自动涌现，无需手排。
- 案例带：K00 独占首行，K01–K06 次行；防御带：簇内最长路径（G01–G04 /
  G05,G06,G08 / G07——单节点层向上合并，容忍一条带内横边）；汇流舱单行。

**S2 虚拟节点**：跨 ≥2 层的边在中间各层插 dummy（占位不渲染），参与排序，
使长边走廊有「预留缝」，出坐标后丢弃（走线仍用正交总线）。

**S3 排序（交叉最小化）**：每簇独立做 barycenter 上下交替扫描 ×4：
下扫用父辈重心、上扫用子辈重心；并列时按语义钉 slot、再按 id。收敛后做一轮
transpose（相邻互换若减少交叉则采纳）。跨簇边参与第二轮全局扫描
（用上一轮的绝对 x 归一化为伪位置）。带区节点按其理论前置的绝对 x 重心排序
（案例/防御卡自动落到自己理论来源的下方）。汇流舱用 slot 钉死叙事序：
C01, C02, C08, C03, C07。

**S4 坐标分配（priority/中位数对齐）**：层内从左到右占格，但每个节点尽力
对齐其同簇父辈的中位 x（单父单子链自动拉直成竖直脊柱）；冲突时按
「度数高者优先」让位，始终保持 I2/I3。簇中心 = 各簇最宽行宽度 + GUTTER 推出。
带区节点在全宽上按重心期望位置排布，同样受最小间隙约束。

**S5 走线车道（lane assignment，tree.js 运行时）**：同一走廊内的多条水平
总线按目标 x 排序，各分配 ±8px 车道偏移，消除总线重叠。

## 3. 验收

构建时自动断言 I1–I5；Playwright 截图逐支逐层目检；
黄支特写必须呈现：主线卡与桥卡不穿插、Y03 落在 Y17 正下方一线。
