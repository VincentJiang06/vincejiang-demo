统一的目录结构与 frontmatter 格式

目录结构 · folder

album-review/
├─ SKILL.md        # 入口 + 流程
├─ rules/         # 确定性规则
├─ scripts/       # 校验器 (.py/.mjs)
├─ schemas/       # JSON Schema
├─ references/    # 参考资料
├─ assets/        # 模板 / 示例
├─ README.md      # 中文说明
├─ README.en.md   # English
└─ CHANGELOG.md   # 版本记录

入口格式 · SKILL.md frontmatter

---
name: album-review
version: 0.1.1
description: >-
  Deep, source-traceable
  long-form 乐评. Use when
  the user names a music
  credit + an album.
  Triggers: "写一篇深度乐评",
  "$album-review".
---

# 正文:锁定决策 + 分步流程
  • SKILL.md 入口:frontmatter(名字 / 版本 / 触发词)+ 分步流程
  • scripts/ 确定性校验器,只用标准库 —— 无需 npm / pip
  • rules/ 把判断标准写死成可复用规则,降随机性
  • schemas/ JSON Schema 校验中间产物,红绿 gate
  • references/ 来源名册等长期参考资料
  • README · .en 中英双语,各 skill 细节看自己的 README

成品Finished products · 拿来即用

课程材料 → 全覆盖、费曼式、可应试的复习笔记。

对事实性问题给出快速、有出处的 BLUF 回答(≤2 / ≤5 分钟)。
humanizer-academic↗
重写 AI 生成的严肃文本(中 / 英 / 混合):去 AI 痕迹同时保留体裁腔调。
mp-cli-sup↗
通过 vince-mp CLI 调试实时运行的微信小程序:持久会话、uid 稳定、免扫码。

微信小程序 Skyline→WebView 迁移,一致性优先,配只读扫描器 + 迁移地图。
编码纪律Coding discipline · 写代码时自动触发
test-driven-development↗
对非平凡行为做 TDD:先写会失败的测试,把测试套件当成当前目标的活规格。
neat↗
会话收尾时把文档 + 跨会话记忆对着代码对账,让知识不腐烂。
循环 & 对抗Loop & adversarial · 工程化自跑
loop-constructor↗
为中大型任务设计工程化循环:分解成带 gate 的子循环树,落盘成可照跑的 runbook。
attacker↗
攻击产品的真实可观测行为:独立 subagent 只记可复现的破坏,与 loop-constructor 配对。
reorganize-logic↗
以代码为唯一事实源重建设计契约层(架构 + 结构 + 接口),删除遗留走评审门。
流水线Meta pipeline · 造 skill 的 skill
skill-conductor↗
带防注水最终验收地驱动 guidance → engineer → zipper 全程。
skill-guidance↗
审计 skill / 仓库并产出 schema 校验的 handoff spec。
skill-engineer↗
从 spec 构建并测试 skill,红-绿-重构,配独立测试组。
skill-zipper↗
为 token 效率、可靠性、触发准确度无损重构现有 skill。
